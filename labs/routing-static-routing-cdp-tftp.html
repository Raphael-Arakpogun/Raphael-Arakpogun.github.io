Lab Overview

This lab demonstrates my ability to configure and verify static routing across multiple network devices using Cisco Packet Tracer. The focus was on IPv4 subnetting, routing logic, neighbor discovery, and validating connectivity between local and remote networks.

Note: This summary reflects my personal learning and does not include assignment instructions or assessment materials.

Scenario

  • Environment: Cisco Packet Tracer
  • Devices: Routers, Layer 3 switch, PCs, and servers
  • Goal: Achieve full end-to-end connectivity using static routes
  • Constraints: Multiple IPv4 subnets and no default routes

What I Built

  • Multiple routed networks connected through routers and a Layer 3 switch
  • Point-to-point WAN links using /30 subnets
  • Remote service network hosting DNS, TFTP, and Web servers
  • Static routes configured on all routing devices

Skills Demonstrated

  • IPv4 addressing and subnetting
  • Static routing configuration
  • Layer 2 vs Layer 3 switch behavior
  • CDP neighbor discovery
  • Connectivity verification using ping and traceroute
  • Saving device configurations to a TFTP server

Testing and Verification

  • Verified connectivity between all directly connected devices
  • Confirmed routing paths using traceroute
  • Validated access to remote network services
  • Confirmed successful configuration uploads to the TFTP server

Troubleshooting Notes

  • Issue: Unable to assign IP address to a switch interface
  • Cause: Interface was operating as a Layer 2 port
  • Resolution: Converted the interface to a Layer 3 port before assigning an IP address

What I Learned

  • Why Ethernet networks require next-hop addresses for static routes
  • The importance of understanding Layer 2 vs Layer 3 switch behavior
  • How routing errors propagate across multiple devices
  • How to systematically verify and troubleshoot routing issues

Next Improvement

A future improvement would be implementing dynamic routing (such as OSPF) to reduce manual configuration and improve scalability.
