Lab Overview

This lab demonstrates my ability to configure and verify static routing across multiple devices in Cisco Packet Tracer. I also reviewed CDP neighbor discovery and practiced backing up device configurations to a TFTP server.

Note: This page reflects my own learning and does not include assignment instructions, answer keys, or assessment materials.

Scenario

  • Environment: Cisco Packet Tracer
  • Goal: Achieve full end-to-end connectivity across multiple routed networks and confirm service reachability.
  • Key Topics: Static routing, CDP verification, and TFTP configuration backup.

What I Built / Configured

  • Configured interfaces (Layer 3 addressing where required)
  • Verified directly connected connectivity before routing changes
  • Created static routes to reach multiple remote networks
  • Used CDP to confirm neighbor relationships and correct physical connections
  • Copied running configurations to a TFTP server for backup

Testing and Verification

  • Interface status: Confirmed key links were up and correctly addressed.
  • Ping testing: Verified pings between directly connected devices first, then across routed networks.
  • Path checks: Used traceroute to confirm traffic followed the expected route.
  • Service checks: Confirmed reachability to remote servers (example: TFTP server ping success).

Troubleshooting Notes

  • Issue: A connectivity test failed after initial routing configuration.
  • Checks performed:
    • Re-checked IP address and subnet mask correctness
    • Verified next-hop addresses for static routes
    • Confirmed interfaces were not shutdown
    • Used CDP to verify neighbor connections were correct
  • Fix: Corrected the addressing/routing mismatch and re-tested until end-to-end pings succeeded.

Skills Demonstrated

  • Static routing (next-hop routing)
  • Subnet awareness and IP planning
  • CDP neighbor discovery and validation
  • Connectivity testing (ping/traceroute)
  • Configuration backup to a TFTP server
  • Structured troubleshooting workflow

What I Learned

  • Why static routes must use the correct next-hop (especially on Ethernet networks)
  • How CDP helps confirm device-to-device connections quickly
  • How to confirm routing success using traceroute, not only ping
  • The value of backing up configs to TFTP after successful verification
