Lab Overview

This lab demonstrates my ability to design and test a basic routed network topology using Cisco Packet Tracer. The focus was on IPv4 addressing, subnetting, and verifying connectivity between multiple networks.

Note: This summary reflects my own learning and does not include assignment instructions or assessment materials.

Scenario

  • Environment: Cisco Packet Tracer
  • Goal: Build a multi-network topology and verify end-to-end connectivity
  • Constraints: Multiple IPv4 subnets, WAN /30 links, and remote services

What I Built

  • Two local LANs connected through routers (RA and RB)
  • Point-to-point WAN link using a /30 subnet
  • Remote network containing DNS, TFTP, and Web servers
  • Structured IPv4 addressing based on subnet requirements

Skills Demonstrated

  • IPv4 addressing and subnetting
  • Understanding of routers, switches, and end devices
  • Basic routing concepts
  • Connectivity testing using ping
  • Basic troubleshooting methodology

Testing and Verification

  • Ping tests between devices on different subnets
  • Verification of IP address and subnet mask configuration
  • Checked physical and logical links in Packet Tracer
  • Confirmed successful access to remote services

Troubleshooting Notes

  • Issue: Initial ping failure between subnets
  • Checks: Verified IP addressing and interface status
  • Fix: Corrected subnet mask on router interface

What I Learned

  • How subnet size impacts address planning
  • The importance of verifying each network hop
  • How small configuration errors affect connectivity

Next Improvement

Extend this topology by adding VLANs and dynamic routing (OSPF) to improve scalability and redundancy.
